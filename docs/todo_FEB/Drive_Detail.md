📁 [PRD 상세 기획] Drive 탭 (테스트 자산 아카이빙)
[목표] 구글 드라이브와 유사한 직관적인 파일 탐색기(File Explorer) UI를 제공하여, 수많은 테스트 케이스를 N-Depth 폴더로 구조화하고 '버전 관리(복제)'를 쉽게 수행할 수 있도록 합니다.

A. 화면 레이아웃 (UI Layout)
Drive 탭은 전형적인 클라우드 스토리지(구글 드라이브, 노션 등)의 인터페이스를 따릅니다.

상단 네비게이션 바 (Breadcrumbs & Actions):

경로 표시 (Breadcrumbs): 현재 위치한 폴더의 경로 표시 (예: 🏠 Home > 2024년 1분기 배포 > 결제 모듈). 상위 경로 클릭 시 해당 폴더로 이동.

액션 버튼 (Primary Actions): [+ 새 폴더], [+ 새 문서(파일)] 버튼 배치.

메인 콘텐츠 영역 (Grid / List View):

현재 폴더 내에 존재하는 **하위 폴더(Folder)**와 **테스트케이스 문서(Document)**들을 아이콘 형태(Grid) 또는 목록 형태(List)로 표시.

폴더는 📁 아이콘, 문서는 📄 아이콘으로 시각적 구분.

우측 속성/정보 패널 (Property Pane - 선택적 노출):

폴더나 문서를 1회 클릭(선택)했을 때, 우측에 해당 항목의 메타 정보(생성일, 수정일, 포함된 케이스 수 등)와 [대시보드/리포트 보기] 버튼을 노출.

B. 주요 엔티티(Entity) 정의 및 역할
📁 폴더 (Folder):

N-Depth(무한 중첩)가 가능한 디렉토리. 물리적인 테스트 케이스를 가지지 않으며, 오직 하위 폴더와 문서를 분류하는 역할만 수행.

📄 문서 (Document):

기존의 '프로젝트(Project)' 단위를 대체하는 핵심 단위. * 1개의 문서 안에 여러 개의 섹션(폴더)과 수백 개의 테스트 케이스(시나리오)가 포함됨.

(예: "로그인 기능 v1.0", "장바구니 결제 흐름")

C. 핵심 기능 및 User Flow
1. 폴더 및 문서 관리 (CRUD)
생성: 현재 위치한 경로에서 새 폴더 또는 새 문서 생성. (이름, 간단한 설명 입력)

수정: 이름 변경 기능 (Rename).

삭제: * 문서 삭제 시: 내부의 모든 테스트 케이스 영구 삭제 (경고 팝업 필수).

폴더 삭제 시: 하위의 모든 폴더 및 문서 연쇄 삭제 (Cascade Delete, 강력한 경고 팝업 필수).

2. 🌟 문서 복제 (버전 관리 / 분기 생성)
기획 의도: 산출물을 보존하고 버전을 업그레이드하기 위한 핵심 기능입니다.

Flow: 특정 문서 우클릭(또는 더보기 메뉴) ➡️ [복사본 만들기 (Duplicate)] 클릭.

동작 로직: * 해당 문서 내부에 있는 모든 섹션과 테스트 케이스들이 완벽하게 동일한 내용으로 새로운 ID를 부여받아 복사됨.

파일명은 [원본 파일명] - 복사본 형태로 자동 생성됨.

활용 시나리오: 기존 "결제 v1.0" 문서를 복제하여 "결제 v2.0"으로 이름을 바꾼 뒤, v2.0 문서 안에서 새로운 케이스를 추가하거나 기존 스텝을 수정함. (원본 v1.0은 그대로 보존됨)

3. 뷰어/에디터 모드 진입 (기존 Manager 기능 연동)
Flow: 📄 문서(Document)를 더블 클릭 (또는 클릭 후 들어가기).

동작 로직: * 기존에 사용하던 2-Pane UI의 TestCaseManager 화면이 열림.

좌측에는 해당 문서에 속한 '섹션(Section)' 트리, 우측에는 '테스트 케이스 리스트 및 상세 편집기'가 기존과 완전히 동일하게 노출 및 작동함.

즉, 사용자는 "어떤 문서를 열 것인가"만 Drive 탭에서 결정하고, 내부 케이스 작성/Import/수정은 완전히 동일한 경험으로 진행.

4. 컨텍스트 대시보드 진입 (Contextual Reporting)
Flow: 폴더 또는 문서 우클릭 ➡️ [대시보드/리포트 보기] 클릭.

동작 로직: * 해당 항목(폴더의 경우 하위 문서 모두 포함)과 연동되어 테스트가 진행되었던 Runner들의 실행 이력과 통계 데이터를 모아서 보여주는 모달 또는 별도 페이지 오픈
